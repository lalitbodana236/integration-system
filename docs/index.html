How the pipeline flows

The platform behaves like an event conveyor belt. The ingestion service accepts the request, Kafka buffers it, processing routes by region, the regional service executes local workflow, and notification fans out delivery.

1 · ingress

ingestion-service

Accepts API requests on /api/po, /api/so, /api/inventory, /api/checklist, /api/location, and /api/media.

2 · backbone

Kafka

Publishes to raw-events and downstream retry / DLQ topics without blocking the HTTP path.

3 · routing

processing-service

Applies validation, duplicate checks, and region routing into region-india, region-us, or region-uk.

4 · workflow

regional-service

Performs regional handling and publishes notification-events for downstream delivery.

5 · fan-out

notification-service

Dispatches EMAIL, SMS, and WEBHOOK notifications with idempotent delivery.
